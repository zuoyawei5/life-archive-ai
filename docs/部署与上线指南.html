一、安卓APK打包

本项目是纯Web应用（HTML+JS+Node.js后端），使用 Capacitor 将Web应用打包为原生Android APK。Capacitor是Ionic团队开发的跨平台原生运行时，可以将Web代码包裹在原生WebView中运行。

1.1 环境准备

必需工具
  • Node.js ≥ 18 LTS（本机已安装）
  • Android Studio（最新版，用于编译APK和调试）— 官方下载
  • JDK 17（Android Studio自带，无需单独安装）
  • Android SDK（通过Android Studio的SDK Manager安装 API 33+）

1.2 打包步骤

1安装Capacitor 
cd life-archive-ai
npm install @capacitor/core @capacitor/cli @capacitor/android --save
npx cap init "AI人生档案馆" "com.lifearchive.app" --web-dir=.

--web-dir=. 表示当前目录就是Web资源目录（因为index.html在根目录）

2添加Android平台 
npx cap add android
npx cap copy android

这会生成 android/ 目录，包含完整的Android项目

3配置Android项目

编辑 android/app/src/main/AndroidManifest.xml，添加必要权限：

<!-- 网络权限 -->
<uses-permission android:name="android.permission.INTERNET" />
<!-- 读取相册（API 33+） -->
<uses-permission android:name="android.permission.READ_MEDIA_IMAGES" />
<!-- 读取相册（API 32及以下） -->
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />

编辑 android/app/src/main/res/values/strings.xml，修改应用名称：

<string name="app_name">AI人生档案馆</string>
<string name="title_activity_main">AI人生档案馆</string>

编辑 android/app/src/main/res/values/styles.xml，设置状态栏颜色：

<style name="AppTheme" parent="Theme.AppCompat.DayNight.NoActionBar">
    <item name="colorPrimary">#0d1117</item>
    <item name="colorPrimaryDark">#0d1117</item>
    <item name="android:statusBarColor">#0d1117</item>
</style>
4配置后端API地址

在 app.js 的API配置区域，将 localhost:3001 改为你的服务器公网IP或域名：

// 开发环境
const API_BASE = window.location.hostname === 'localhost' 
  ? 'http://localhost:3001' 
  : 'https://你的域名.com';  // 生产环境用HTTPS
5生成Debug APK 
cd android
.\gradlew assembleDebug

生成的APK在：android/app/build/outputs/apk/debug/app-debug.apk

Debug APK可以直接安装到手机测试，但不能上架应用商店

6生成Release签名APK

首先生成签名密钥：

keytool -genkey -v -keystore life-archive.keystore -alias lifearchive -keyalg RSA -keysize 2048 -validity 36500

按提示输入密码和信息（密钥有效期100年）

在 android/app/build.gradle 中配置签名：

android {
    signingConfigs {
        release {
            storeFile file('../../life-archive.keystore')
            storePassword '你的密码'
            keyAlias 'lifearchive'
            keyPassword '你的密码'
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
        }
    }
}

生成Release APK：

cd android
.\gradlew assembleRelease

生成的APK在：android/app/build/outputs/apk/release/app-release.apk

7生成AAB格式（应用商店要求） 
cd android
.\gradlew bundleRelease

生成的AAB在：android/app/build/outputs/bundle/release/app-release.aab

Google Play要求上传AAB格式，国内应用商店接受APK或AAB

2.2 必需资质清单

上架前准备
  1. 软件著作权：在中国版权保护中心申请，约2-3个月下证（可加急）。费用约300-800元
  2. 企业资质：营业执照（个人开发者部分商店也接受，但限制较多）
  3. ICP备案：如果APP有后端服务，域名需要ICP备案
  4. 应用图标：512×512px PNG
  5. 应用截图：至少5张，1080×1920px
  6. 隐私政策：必须提供隐私政策URL，说明数据收集和使用方式

2.3 上架流程（以华为应用市场为例）

1注册华为开发者账号（https://developer.huawei.com），完成实名认证
2进入"应用发布" → 创建应用 → 填写基本信息（应用名称、简介、分类等）
3上传APK/AAB文件 → 系统自动检测应用信息
4填写版本信息 → 上传截图和图标 → 填写隐私政策URL
5提交审核 → 等待3-5个工作日 → 审核通过后自动上架

3.2 部署步骤

1连接服务器 
# 使用SSH连接（Windows用PowerShell，Mac/Linux用Terminal）
ssh root@你的服务器公网IP

# 首次连接输入yes确认，然后输入密码
2安装Node.js环境 
# 安装Node.js 20 LTS
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
apt-get install -y nodejs

# 验证安装
node -v   # 应显示 v20.x.x
npm -v    # 应显示 10.x.x

# 安装PM2（进程管理，保持应用持续运行）
npm install -g pm2
3安装Nginx（反向代理） 
apt-get update
apt-get install -y nginx
4上传项目代码 
# 方法1：使用scp上传（在本地PowerShell执行）
scp -r ./life-archive-ai root@你的服务器IP:/opt/

# 方法2：使用Git（推荐）
# 在服务器上：
cd /opt
git clone 你的仓库地址 life-archive-ai

# 安装依赖
cd /opt/life-archive-ai
npm install --production
5配置环境变量 
# 创建环境变量文件
cat > /opt/life-archive-ai/.env << 'EOF'
ZHIPU_API_KEY=你的智谱API密钥
PORT=3001
NODE_ENV=production
EOF

# 安全权限
chmod 600 /opt/life-archive-ai/.env
6使用PM2启动应用 
cd /opt/life-archive-ai
pm2 start server.js --name "life-archive"
pm2 save
pm2 startup  # 设置开机自启

此时应用在 http://服务器IP:3001 运行

7配置Nginx反向代理 
cat > /etc/nginx/sites-available/life-archive << 'EOF'
server {
    listen 80;
    server_name 你的域名.com;  # 或服务器IP

    # 前端静态文件
    location / {
        root /opt/life-archive-ai;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF

ln -s /etc/nginx/sites-available/life-archive /etc/nginx/sites-enabled/
nginx -t          # 测试配置
systemctl reload nginx
8配置HTTPS（必须） 
# 安装Certbot
apt-get install -y certbot python3-certbot-nginx

# 申请免费SSL证书（Let's Encrypt）
certbot --nginx -d 你的域名.com

# 按提示确认，自动配置HTTPS
# 证书自动续期已配置
注意
使用HTTPS前，必须先完成域名ICP备案（国内服务器）和域名解析（DNS A记录指向服务器IP）
9配置防火墙 
# 阿里云安全组规则（在控制台配置）：
# - 开放 80 端口（HTTP）
# - 开放 443 端口（HTTPS）
# - 开放 22 端口（SSH）
# - 不需要开放 3001 端口（通过Nginx代理）

# 服务器防火墙
ufw allow 80
ufw allow 443
ufw allow 22
ufw enable
10修改前端API地址

在服务器上编辑 app.js，将API基础地址改为生产环境：

// app.js 顶部配置区域
const API_BASE = 'https://你的域名.com';

或在Nginx中配置相对路径代理，前端用 /api/ 相对路径

3前端登录后自动同步 
// app.js doLogin函数中添加
async function doLogin() {
  // ... 原有登录逻辑 ...
  
  // 从云端拉取用户数据
  try {
    const resp = await fetch(`${API_BASE}/api/user/data/${phone}`);
    const result = await resp.json();
    if (result.success && result.data) {
      // 恢复云端数据
      if (result.data.photos) {
        localStorage.setItem('lifeArchivePhotos', result.data.photos);
      }
      if (result.data.memoirs) {
        localStorage.setItem('lifeArchiveMemoirs', result.data.memoirs);
      }
    }
  } catch(e) { console.error('云端同步失败', e); }
  
  // ... 继续原有逻辑 ...
}

// 照片分析完成后自动同步到云端
async function syncToCloud() {
  if (!currentUser) return;
  try {
    await fetch(`${API_BASE}/api/user/sync`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        phone: currentUser,
        photos: localStorage.getItem('lifeArchivePhotos'),
        memoirs: localStorage.getItem('lifeArchiveMemoirs'),
        settings: { theme: localStorage.getItem('lifeArchiveTheme') }
      })
    });
  } catch(e) { console.error('同步失败', e); }
}

阿里云短信服务接入（替换模拟验证码）

// server.js 接入阿里云短信SDK
const dysmsapi = require('@alicloud/dysmsapi20170525');
const OpenApi = require('@alicloud/openapi-client');

const smsClient = new dysmsapi.default(new OpenApi.Config({
  accessKeyId: '你的AccessKeyId',
  accessKeySecret: '你的AccessKeySecret',
  endpoint: 'dysmsapi.aliyuncs.com'
}));

// 发送验证码
app.post('/api/sms/send', async (req, res) => {
  const { phone } = req.body;
  const code = String(Math.floor(100000 + Math.random() * 900000));
  
  // 存储验证码（5分钟过期）
  verifyCodes.set(phone, { code, expire: Date.now() + 300000 });
  
  // 调用阿里云短信发送
  const result = await smsClient.sendSms(new dysmsapi.SendSmsRequest({
    phoneNumbers: phone,
    signName: '人生档案馆',
    templateCode: 'SMS_XXXXXX',  // 你的短信模板Code
    templateParam: JSON.stringify({ code })
  }));
  
  res.json({ success: true });
});
